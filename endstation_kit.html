1 Main Objectives

  1. Terminate photon delivery system with an appropriately sized Be window
  2. Gantry to support vacuum pressure of about 700 pounds on the Be window. This gantry should move vertically with the table to track the various beams.
  3. BPM follows Be window
  4. The main table is something like 2.5 m by 1.5 m and has 4 degrees of freedom:
    • Vertical to follow beam height
    • Pitch to make surface of table parallel to beam
    • Yaw and horizontal to align table holes to beam path
  5. Rail on table serves two purposes:
    • Fixed height supports for ion chambers (table pitch and vertical ensures ion chambers are in beam path)
    • Support for transport pipe when goniometer is in use
  6. Transport pipe for delivering beam to goniometer.

1.1 Be window

I am assuming that this is a thin window which is a bit bigger than the beam. It gets positioned vertically into the beam (with a bit of wiggle room for adjusting the mirrors).

An alternative would be to have a window enough to accommodate the full vertical range of the beam. Presumably, this would have to be significantly thicker.

1.2 Be window gantry

The gantry must support a vacuum force in the negative beam direction and adjust vertically accroding to the beam height.

Options:

  • Gantry is mounted on the main table, as main table moves to meet the beam, the window moves with it.
  • Gantry sites on an independent table whose motion is coordinated with the main table motion.

1.3 BPM

This is, presumably, FMBO's nanoBPM.

1.4 Table

See "market research" from Axilon

  • Optical table supported on a granite block.
  • Articulation of all degrees of freedom.
  • Need to define margins for vertical, pitch, horizontal, and roll.
  • In many cases, table will not be level, so a rim will be needed around the downstream end of the table.

1.5 Rail system

The base of this can be standard X-rail. The vertical supports for both the ion chambers and the transport pipe need design work.

The ion chamber supports should be fixed. When the table is at a height and pitch appropriate to the mirror positions, the ion chambers will be in the beam.

The transport pipe support is a bit more complicated. It should have some flexibility in pitch such that, along with the bellows attaching it to the entry window, its is possible to clamp the bellows in place and adjust the height of the downstream support.

1.5.1 Downstream support

Motorized jack to lift/lower the downstream end of the pipe. This rotation is supported by a hinged upstream support and a bellows connecting to the entry window.

How is the height of the jack set? My suggestion is that a series of survey marks is placed on a nearby wall. A survey target is placed on the transport pipe just above the jack. A theodolite is set to the selected beam height. Looking through the theodolite, use a hand-held controller to actuate the jack.

Given the size of the transport pipe and the possibility for damage if the jack is driven incorrectly, this is a good candidate for this sort of non-automated solution. That is, a person must be physically present to do the transport pipe height adjustment.

The exit window should be a generous enough in height so that the mirrors can be tweaked without needing to readjust the jack height.

1.6 Transport pipe

  • Pipe with fittings on top side for lifting by overhead hoist.
  • Supported at one point on the table rail. This support is hinged in the pitch direction to facilitate placement of transport pipe and adjustment of angle.
  • Supported at downstream end by adjustable height pillar so pitch of transport pipe can be set appropriate to beam mode.
  • Quick connect to upstream window flange via bellows. (So, not UHV)
  • Be or plastic window on downstream end. Perhaps some kind of snout for bringing beam close to eucentric of goniometer.
  • Pumping port. Rough + turbo.
  • Will need clear procedure for placement of pipe at a specific angle and for adjustment of angle when changing modes.

2 Other features

Here is a list of additional toys, widgets, and features I would like to have for the end station.

2.1 Gas handling

Automated system for rough-pumping ion chambers and refilling with new gases. (Get details from, e.g., B18 at Diamond.)

2.2 Camera

See https://www.dectris.com/pilatus-100k.html#main_head_navigation. The Pilatus 100K is quite cheap, but not for long.

Many, many uses:

  • Beam profile characterization at arbitrary positions along beam path
  • Area detector for goniometer
  • Yoneda-EXAFS
  • bulk-XES rig

2.3 Stages I'd like to have

  • Three 4-axis stacks (x,y,pitch,yaw)
    • 2 for 4-channel ion chambers
    • 1 for polycap lens

    I would like permanent mounting stages for these instruments so that they can become regular fixtures at the beamline.

  • Rotation stage for samples – disk bolted to rotation, all standing on a two-axis stage. A rotation stage with a disk can easily mount 20 or more sample. Translation between samples can be made without needing to move the fluorescence detector (the caveat is that there must be adequate space in front of the detector for the samples to pass by). My thought is to make many (50 or so) plates and loan them to visitors for mounting samples at home.
  • Rotation stage for reference foils – disk bolted to rotation, all standing on a two-axis stage. This allows a change of reference foil to be easily automated.
